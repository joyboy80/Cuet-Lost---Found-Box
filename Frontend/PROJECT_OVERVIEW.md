# My Work and Project Overview

## What I Am Doing

I am building a web-based **Lost and Found Management System** for CUET (Chittagong University of Engineering & Technology).

Right now, I am working on the **frontend implementation** with responsive pages, form handling, search/filter features, role-based dashboards, and admin management flows.

My current development focus includes:
- Improving user experience across all pages (mobile + desktop)
- Validating report forms for lost/found items
- Managing item data in browser LocalStorage for demo purposes
- Building role-based functionality for users, admins, and super admins
- Preparing the project for backend integration in the next phase

## Project Overview

**Project Name:** CUET Lost & Found Box

CUET Lost & Found Box is a centralized platform that helps students and staff:
- Report lost items
- Report found items
- Search and filter submitted items
- Track report status
- Connect with the relevant reporter for recovery

### Current Version

- Version: `v1.0` (Frontend-focused)
- Data Storage: Browser LocalStorage
- Backend: Planned for `v2.0`

### Core Features

- Home page with project highlights and recent items
- Authentication (login and registration)
- Lost item reporting form
- Found item reporting form
- Search page with filters and view modes
- Community feed for latest updates
- User dashboard for report tracking
- Admin panel for approve/reject moderation
- Super admin panel for user/admin management

### Technology Stack

- HTML5
- CSS3
- Vanilla JavaScript
- LocalStorage (temporary data persistence)

### Planned Next Step

Integrate a backend API (Django REST Framework or FastAPI) with a database (PostgreSQL) and secure JWT-based authentication.
